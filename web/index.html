Verifiable memory for AI agents

Know what your agent actually did. And prove it's true.

Ask an agent what it did and you cannot trust the answer, it invents a confident history or forgets the thread. AgentMem keeps a record grounded in your files and your git history, and verifies the account against the repository, not the agent's word.

How it works

01 · capture

Record, out of band

Actions and their real effects are captured at the tool boundary, never scraped from what the agent says about itself.

02 · verify

Check against ground truth

Every claim is checked against the repository and git, which decide. The model never grades itself; the agent is an untrusted witness.

03 · surface

Hand it back when it matters

After a compaction, a new session, or on request, the verified record returns, grounded and selective. Most of the time it stays silent.

We also publish what did not work. AgentMem does not claim to make your agent code faster, that came back a null and we say so. What it earns is an honest, verifiable account of what happened. In a field of inflated benchmarks, that is the point.

Using Claude Code

Install from the marketplace

/plugin marketplace add AgentMem/agentmem
/plugin install agentmem@agentmem
/agentmem:setup

Three commands, typed right in Claude Code, no terminal. The plugin fetches its own engine on first use and the /agentmem:setup wizard handles your API key. From then on Claude Code keeps a verified record as you work, and speaks up only when it matters.

Where this is going

Today AgentMem is a flight recorder for one agent. The same record is built to be shared. When several agents, or agents and people, work together, they read one verified account of who did what, and trust it because it is checked against reality rather than taken on each other's word. Memory stops being a booster for a single agent's thinking and becomes the source of truth a system of agents coordinates on. That is the substrate autonomous work will need.
